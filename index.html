FOSS projects


sflag : golang cmdline arg parser using struct syntax

qslinux/blkid : extracts useful info from that command on linux

qslinux/df : extracts useful info from that command on linux

qslinux/dmidecode : extracts useful info from that command on linux

qslinux/etcfstab : extracts useful info from that file on linux

qslinux/etchosts : extracts useful info from that file on linux

qslinux/etcservice : extracts useful info from the service command on linux

qslinux/etcshadow : extracts useful info from that file on linux

qslinux/etcuser : extracts useful info from that file on linux

qslinux/hp : extracts useful info, using the hpaucli program, about all hp disk controllers on a linux system

qslinux/md : extracts useful info from the mdadm command on linux

qslinux/nmap : extracts mainly up/down status of network devices, as seen from linux.

qslinux/parted : extracts useful info from that command on linux

qslinux/scsi : extracts useful info from the lsscsi on linux

qslinux/smartctl : extracts smartctl information on linux (hp and supermicro are handled better)

qslinux/tgtd : extracts useful info about iscsi targets provided by the linux server on which it runs.

qcfg : custom config file reader/writer

genutil : collects in place the utility functions used in various golang scripts

gencsv : generates a package specific to the hcsv file format described in its input

qsdata2 : collects information about data storage from various commands on a linux box, and outputs it in csv-formatted file

qsdataetc : collects info from various sources on a linux box - mainly etc files, but also some commands, and outputs it into csv-formatted file

qsdatanet : runs nmap to detect up/down status of network devices on a configured subnets, and produces csv-formatted file output

  • Operations monitoring and troubleshooting. Dev-Ops work ratio is 50:50.

  • Participate in regular deployments of code updates.
  • Learn on the job and explore new technologies with little supervision.
  • Development work on open source projects like Libreoffice/Thunderbird based on client requirement.

Requirements

  • Undergraduate degree in Computer Science or equivalent work experience.
  • Good coding skills in C++/Golang with 0-2 years experience.
  • Good knowledge of GNU/Linux operating system.
  • Ability to research technical problems and solutions online.
  • Excellent communication skills.
  • Willing to work in any shifts within 4:00 AM IST to 11:00 PM IST.

  • Salary based on experience at par with industry standards.
